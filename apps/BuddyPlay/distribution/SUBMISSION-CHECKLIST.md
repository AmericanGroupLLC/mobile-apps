## Required GitHub Actions secrets

These are referenced by `.github/workflows/release.yml` (and triggered on a `v*` tag push). Until set, `release.yml` will fail to produce signed artefacts.

### Common (every repo)

  - **`ANDROID_KEYSTORE_BASE64`** (Android signing) — `base64 -w 0 upload-keystore.jks`
  - **`ANDROID_KEYSTORE_PASSWORD`** (Android signing) — chosen at keystore creation
  - **`ANDROID_KEY_ALIAS`** (Android signing) — chosen at keystore creation (commonly `upload`)
  - **`ANDROID_KEY_PASSWORD`** (Android signing) — chosen at keystore creation
  - **`APP_STORE_CONNECT_API_ISSUER_ID`** (iOS upload) — App Store Connect -> Users and Access -> Integrations -> App Store Connect API -> Issuer ID (UUID)
  - **`APP_STORE_CONNECT_API_KEY_ID`** (iOS upload) — same screen, the 10-char Key ID for the .p8
  - **`APP_STORE_CONNECT_API_KEY_P8_BASE64`** (iOS upload) — `base64 -w 0 AuthKey_XXXXXXXXXX.p8`
  - **`APPLE_TEAM_ID`** (iOS signing) — developer.apple.com -> Membership -> Team ID (10-char)
  - **`MATCH_GIT_URL`** (iOS signing) — HTTPS URL to a private git repo for fastlane match (recommend creating `AmericanGroupLLC/match-certificates`)
  - **`MATCH_PASSWORD`** (iOS signing) — passphrase you choose to encrypt the match repo
  - **`PLAY_STORE_PACKAGE_NAME`** (Play upload) — this app's Android applicationId (see below)
  - **`PLAY_STORE_SERVICE_ACCOUNT_JSON`** (Play upload) — Play Console -> Setup -> API access -> create service account, download JSON, paste FULL JSON content

### Optional (CI coverage)

  - **`CODECOV_TOKEN`** (optional) — codecov.io repo settings (optional - coverage upload only)

## Material the user must provide

These cannot be generated by an automated agent — they require an Apple/Google account login or designer input.

### Apple side

1. **Apple Developer Account** ($99/yr) for `AmericanGroupLLC` — confirm enrolment status at <https://developer.apple.com/account>.
2. **App Store Connect record** for bundle `com.americangroupllc.buddyplay` — register the app at <https://appstoreconnect.apple.com> → My Apps → `+`.
3. **App Store Connect API key** — Users and Access → Integrations → App Store Connect API → `+` → role *App Manager*. Download the `.p8` file (it can only be downloaded once). Copy the Issuer ID and Key ID.
4. **Distribution certificate + provisioning profiles** — easiest path is to run `fastlane match init` once locally (creates `AmericanGroupLLC/match-certificates` private repo), then `fastlane match appstore` for each bundle ID listed above. The MATCH_GIT_URL secret then points CI at this repo.
5. **Apple Team ID** — 10-char alphanumeric from <https://developer.apple.com/account/#!/membership>.

### Google side

1. **Google Play Console account** ($25 one-time) for `AmericanGroupLLC`.
2. **Play Console app record** for package `com.americangroupllc.buddyplay` — Play Console → `+ Create app`.
3. **Upload keystore** — generated locally with `keytool -genkey -v -keystore upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload`. Hand over the `.jks` file + the alias + both passwords; the agent will base64-encode and `gh secret set`.
4. **Service account for upload API** — Play Console → Setup → API access → create new service account → grant *Release manager* role on this app → download JSON key.

### App content

5. **Marketing copy** — short description (80 chars), full description (4000 chars), what's-new (500 chars), keywords (100 chars). Drafts will be generated in `distribution/app-store/metadata/<locale>/` and `distribution/play-store/metadata/<locale>/` in Phase 5; user reviews and approves.
6. **Marketing screenshots** — at least 3 per device class (iPhone 6.7", 6.5", 5.5"; iPad 12.9" Gen 2 + Gen 6+; Pixel phone; 7" tablet; 10" tablet). The agent can scaffold a fastlane snapshot config; the user runs it on a real device or simulator with realistic data.
7. **App icon** — final 1024×1024 (iOS) and 512×512 (Play feature graphic 1024×500). The agent cannot design these.
8. **Privacy answers** — App Privacy "nutrition label" (Apple) and Data Safety form (Google). Drafts will be generated in `distribution/{app-privacy.json,data-safety.json}` from a static analysis of which SDKs collect what; user verifies every line and clicks submit in the consoles.

## Wiring sequence (Phase 4 — once material is in hand)

`sh
# Android signing (this app)
base64 -w 0 upload-keystore.jks | gh secret set ANDROID_KEYSTORE_BASE64 --repo AmericanGroupLLC/BuddyPlay
echo -n "<password>"             | gh secret set ANDROID_KEYSTORE_PASSWORD --repo AmericanGroupLLC/BuddyPlay
echo -n "upload"                 | gh secret set ANDROID_KEY_ALIAS         --repo AmericanGroupLLC/BuddyPlay
echo -n "<password>"             | gh secret set ANDROID_KEY_PASSWORD      --repo AmericanGroupLLC/BuddyPlay

# Play upload
echo -n "com.americangroupllc.buddyplay"   | gh secret set PLAY_STORE_PACKAGE_NAME --repo AmericanGroupLLC/BuddyPlay
gh secret set PLAY_STORE_SERVICE_ACCOUNT_JSON --repo AmericanGroupLLC/BuddyPlay < play-service-account.json

# iOS signing (every Apple-related secret can be the same value across all 6 repos if same Apple Team)
base64 -w 0 AuthKey_XXXXXXXXXX.p8 | gh secret set APP_STORE_CONNECT_API_KEY_P8_BASE64 --repo AmericanGroupLLC/BuddyPlay
echo -n "<key id>"                | gh secret set APP_STORE_CONNECT_API_KEY_ID         --repo AmericanGroupLLC/BuddyPlay
echo -n "<issuer uuid>"           | gh secret set APP_STORE_CONNECT_API_ISSUER_ID      --repo AmericanGroupLLC/BuddyPlay
echo -n "<10-char team>"          | gh secret set APPLE_TEAM_ID                        --repo AmericanGroupLLC/BuddyPlay
echo -n "https://github.com/AmericanGroupLLC/match-certificates.git" | gh secret set MATCH_GIT_URL --repo AmericanGroupLLC/BuddyPlay
echo -n "<passphrase>"            | gh secret set MATCH_PASSWORD                       --repo AmericanGroupLLC/BuddyPlay
`

After secrets are set:

`sh
gh secret list --repo AmericanGroupLLC/BuddyPlay
`

…should show every name listed in *Required GitHub Actions secrets* above.

## Cut a release (Phase 6)

After every secret is in place:

`sh
cd BuddyPlay
# bump versionCode in android/app/build.gradle.kts and CFBundleVersion in ios/...
git add -A && git commit -m "chore(release): bump to vX.Y.Z"
git push
git tag vX.Y.Z && git push origin vX.Y.Z
gh run watch --repo AmericanGroupLLC/BuddyPlay \ --exit-status
`

On success `release.yml` produces `.aab` (Play) and `.ipa` (App Store) and uploads to Play Internal Testing + TestFlight.
